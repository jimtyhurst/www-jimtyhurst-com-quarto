Queued Transaction

Problem

How can a client application organize a set of domain model changes to be submitted in a transaction where there are many clients adding new independent data?

Context

tbd

Forces

  • Length of transaction.

  • Objects cannot be modified outside of a transaction.

  • Signal to abort.

  • Transaction modes supported by the application server.

  • Failure to commit.

Solution

Specify the actions to be performed in a transaction and submit that specification into the queue of a transaction processor.
